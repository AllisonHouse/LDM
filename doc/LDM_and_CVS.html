CVS and LDM Development


2004-04-21

Structure of the CVS Repository

The CVS repository of the LDM project comprises two subdirectory hierarchies: "src" (which contains source-code) and "doc" (which contains documentation).

The file "src/VERSION" contains the version identifier of the LDM package.

The file "src/HISTORY" contains manually-added descriptions of changes to the LDM package.

Naming Conventions for CVS Release Tags

CVS tags for  package releases are named
ldm-major_minor[_bugfix]
where:
major
Is the major release number
minor
Is the minor release number
bugfix
Is the optional bug-fix number.  The brackets denote optionality and are not part of the tag.

CVS Policy Terms

The main trunk is for formal releases only.

All development shall take place on CVS branches.  For example, to add the ability to filter data-products by "allowed" product-identifiers to upstream LDMs based on LDM version 6.0.15, a developer could do the following in an empty directory:
$ export CVSROOT=...
$ cvs rtag -b -r ldm-6_0_15 upstream_filtering doc src
...
$ cvs co -r upstream_filtering doc src
...
$ cd src
$ <<modify and test>>
$ cd ../doc
$ <<modify and vet>>
$ cd ..
$ cvs commit -m "Added upstream filtering by product-ID"
...
There are other ways of creating branches in CVS.  More information on CVS can be found at http://www.cvshome.org/

Only the Project Manager may merge changes on a branch onto the main trunk.  Developers shall notify the Project Manager when they believe their modifications are ready to be incorporated into the main trunk.

Write access may be terminated by the Project Manager for violating this policy.
